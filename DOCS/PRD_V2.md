# PRD Version 2

In this document, we index the PRDs for the second version of the product.

## PRD Index

- [PRD for Workspace Convention](/docs/RECURSIVE_WORKSPACE_CONVENTION_PRD.md)
  Goal: Define the recursive filesystem convention for v2 workspaces, including routed structure, ancestry-based visibility, artifact-local context, and first-class reference placement.
- [PRD for Data Model](/docs/RECURSIVE_WORKSPACE_DATA_MODEL_PRD.md)
  Goal: Define the distributed v2 data model for recursive workspaces, including artifact contracts, colocated sidecars, runtime assembly, version selection, references, prompts, dependencies, validation, provenance-oriented metadata, and fingerprinting.
- [PRD for Workflow Model](/docs/WORKFLOW_MODEL_PRD.md)
  Goal: Define the v2 workflow model, including the replacement of `STATUS` with `WORKFLOW`, stage and node structure, dependencies, output artifacts, status semantics, recursive workflow composition, and managed workflow hydration rules.
- [PRD for Iteration, Variants, and Visual Edit History](/docs/ITERATION_VARIANTS_AND_VISUAL_EDIT_HISTORY_PRD.md)
  Goal: Define how artifacts are iteratively generated, reviewed, regenerated, versioned, compared, selected, and edited over time, including human-in-the-loop review, automated review, consistency checks, selected variant promotion, undo/redo support, and event-based visual change history for future AI analysis.
- [PRD for Editorial and Cut Semantics](/docs/EDITORIAL_AND_CUT_SEMANTICS_PRD.md)
  Goal: Define the v2 editorial model for transitions and cut types between shots so sequence planning, shot outputs, and final assembly share a consistent understanding of cinematic joins.
- [PRD for Context Engineering](/docs/CONTEXT_ENGINEERING_PRD.md)
  Goal: Define how agent context is assembled in v2, including workspace memory, idea and canon files, stable system prompting, tool affordances derived from the data model, and the boundaries of local versus inherited context.
